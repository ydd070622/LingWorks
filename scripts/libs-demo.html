Framer Motion 动画引擎
让弹窗、卡片、面板的进出有物理感（弹簧、缓动），而不是生硬的 display:none/block 切换。
提升点：① 客户弹窗滑出更流畅 ② 卡片 hover 有弹性上浮 ③ tab 切换淡入 ④ 拖拽时有跟随感

sonner 通知提示
替代现在的 alert() 和 confirm()。操作后右上角弹出小卡片提示，不打断流程。
提升点：① 保存客户后弹"已保存"② 删除前弹确认 ③ 标记收款后弹"已收款 ¥8,400" ④ 错误提示不打断操作
现状：alert 弹窗（阻塞 + 难看）
引入 sonner：右上角卡片 不阻塞
💡 差异：左边的 alert 是浏览器原生弹窗，必须点确定才能继续。右边的 sonner 提示 3 秒后自动消失，不打断操作。多个提示会堆叠。
recharts 数据图表
Dashboard 的漏斗图和排行现在是纯 CSS 条形，recharts 能做出真正的交互式图表。
提升点：① 转化漏斗变可交互（hover 显示转化率）② 月度成交趋势折线图 ③ 回款进度环形图 ④ 客户来源饼图
现状：CSS 条形（静态）

date-fns 日期处理
替代 helpers.ts 里手写的 daysDiff/fmtDate/today。
提升点：① "3天前"/"明天"/"逾期2天"自动算 ② 日期格式统一 ③ 处理时区/跨月不再出错 ④ 跟进日期显示更人性化
现状：手写函数（简陋）
fmtDate('2026-06-15') → "6月15日"
daysDiff(a, b) → -2
fuDisplay(date) → "逾期2天"
⚠️ 只能输出固定格式
⚠️ 没有"3天前"/"刚刚"/"下周三"这种相对时间
⚠️ 跨月/跨年时容易算错
⚠️ 函数都是自己写的，没经过测试
引入 date-fns（专业）
formatDistanceToNow(date) → "3天前"
format(date, 'M月d日 EEEE') → "6月15日 星期日"
differenceInCalendarDays → -2
isToday / isTomorrow / isYesterday
✅ "刚刚"/"3分钟前"/"2小时前"/"昨天"
✅ "下周三跟进"/"3天后到期"
✅ 跨月跨年自动正确
✅ 200+ 个日期函数，按需引入
💡 这个提升最"隐形"——用户不会直接看到，但跟进时间线、工作台提醒、合同签约日期的显示都会更准确、更人性化。
总结：该不该引入？
✅ 推荐引入
sonner极低成本替代 alert，体验立刻变好
date-fns极低成本日期显示更准更人性化
Framer Motion中等成本动画质感提升明显
recharts中等成本Dashboard 数据可视化
⚠️ 引入成本
• 每个库需要 npm install
• 需要逐个组件替换调用点
• sonner/date-fns 改动小（几小时）
• Framer Motion/recharts 改动中等（1-2天）
• 不影响现有功能，可逐步引入
